git checkout develop
git checkout -b fix/commits
git rebase -i HEAD~3
# В редакторе: оставить 'pick' у первого коммита, остальные заменить на 'squash' или 's'
# Сохранить и закрыть редактор (в Vim: Esc → :wq)
# Откроется окно изменения сообщения объединённого коммита — внести новое сообщение
# Сохранить и закрыть редактор
git push --force-with-lease origin fix/commits

1. Как просмотреть изменения перед push после rebase?
После выполнения git rebase, перед отправкой изменений на удалённый репозиторий (git push),
 можно посмотреть разницу с удалённой веткой с помощью команды: git diff origin/main..HEAD
Или посмотреть историю коммитов: git log --oneline --graph --all
2. Чем отличается --force от --force-with-lease?
--force Просто перезаписывает удалённую ветку без проверок
--force-with-lease Перед перезаписью проверяет, не были ли изменения на удалённой ветке со стороны других пользователей
3. Как отменить rebase, если что-то пошло не так?
Если ещё не завершили rebase , просто прервать его: git rebase --abort
Если rebase уже успешно выполнен , используйте reflog, чтобы найти предыдущее состояние: git reflog
Найти хэш коммита до начала rebase , затем : git reset --hard <хэш_коммита>
